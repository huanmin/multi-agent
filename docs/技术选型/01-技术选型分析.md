### 3.2 移动端开发策略

#### 阶段 1: MVP（桌面优先）

**时间**: 前 8 周

**策略**:
- 专注桌面端开发（Windows/macOS/Linux）
- 前端使用响应式设计（为移动端预留适配空间）
- 后端接口设计考虑移动端复用

**前端响应式要点**:
```typescript
// 使用 Tailwind 响应式类
<div className="w-full md:w-80 lg:w-96">  {/* 侧边栏宽度 */}
<div className="hidden md:block">         {/* 桌面显示，移动端隐藏 */}
<div className="flex-col md:flex-row">    {/* 移动端垂直，桌面水平 */}
```

#### 阶段 2: 移动端适配

**时间**: v1.2（第 16 周后）

**策略**:
- 添加移动端特定 UI（底部导航、手势操作等）
- 优化移动端体验（消息气泡、输入框等）
- 使用 Tauri 构建 iOS/Android 应用

**移动端特有调整**:
```
┌─────────────────────────────────┐
│         Mobile UI               │
├─────────────────────────────────┤
│  • 底部导航栏（微信风格）        │
│  • 手势返回                     │
│  • 消息左滑操作                 │
│  • 软键盘适配                   │
│  • 屏幕旋转适配                 │
└─────────────────────────────────┘
```

### 3.3 移动端技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **UI 框架** | React + Tailwind | 响应式适配 |

### 4.2 模型配置数据结构

```typescript
// 模型提供商配置
interface ModelProvider {
  id: string;                    // 唯一标识
  name: string;                  // 显示名称
  type: 'official' | 'custom';   // 官方内置 or 用户自定义

  // API 配置
  baseUrl: string;               // API 基础地址
  apiKey: string;                // API 密钥（加密存储）

  // 模型列表
  models: ModelConfig[];

  // 请求配置（遵循 OpenAI 格式）
  requestConfig: {
    headers?: Record<string, string>;
    timeout: number;             // 默认 30s
    maxRetries: number;          // 默认 3
  };
}

// 单个模型配置
interface ModelConfig {
  id: string;
  name: string;                  // 显示名称
  modelId: string;               // API 调用时的 model 参数

  // 能力参数
  maxTokens: number;
  contextWindow: number;         // 上下文长度
  supportsStreaming: boolean;    // 是否支持流式
  supportsFunctionCalling: boolean; // 是否支持工具调用

  // 成本（用于 Dashboard 统计）
  pricing: {
    inputPrice: number;          // $/1K tokens
    outputPrice: number;         // $/1K tokens
  };

  // 自定义请求体字段（兼容非标准 API）
  customFields?: Record<string, any>;
}
```
