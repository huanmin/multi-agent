# 术语表

## Harness Engineering 术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 束缚工程 | Harness Engineering | 一种系统化的 AI 开发方法论，包含三类工程实践 |
| 上下文工程 | Context Engineering | 管理和维护 AI 系统上下文信息的实践 |
| 架构约束 | Architectural Constraints | 通过自动化工具强制执行架构规则的实践 |
| 垃圾回收 | Garbage Collection | 自动清理过期和冗余资源的实践 |
| 园丁代理 | Gardener Agent | 负责代码库维护的自动化智能体 |

## 架构术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 分层架构 | Layered Architecture | 将系统按职责分为多个层次的架构模式 |
| 表现层 | Presentation Layer | 处理用户交互和 API 接口的层 |
| 应用层 | Application Layer | 协调用例和业务流程的层 |
| 领域层 | Domain Layer | 包含核心业务逻辑的层 |
| 基础设施层 | Infrastructure Layer | 提供技术支持的层 |

## 可观测性术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 指标 | Metrics | 系统状态的量化测量值 |
| 追踪 | Traces | 请求在系统中流转的完整路径 |
| 日志 | Logs | 系统事件的离散记录 |
| 可观测性 | Observability | 通过外部输出推断系统内部状态的能力 |

## 智能体术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 智能体 | Agent | 能够自主执行任务的 AI 实体 |
| 工具 | Tool | 智能体可以调用的外部能力 |
| 提示工程 | Prompt Engineering | 设计和优化 AI 提示的技术 |
| 上下文窗口 | Context Window | AI 模型一次能处理的最大文本量 |
